# おひさまポータル（公開用）

架空のメンバーとサンプルコンテンツだけで構成した、公開用ポータルです。  
このREADMEは、画面機能の説明だけでなく、`ohisama-backend` と `ohisama-portal` で実装しているデータ運用の考え方を説明することを目的にしています。

公開用のため、データはすべて架空です。

## このREADMEで伝えたいこと

1. どのようにデータを取得しているか
2. どのようにデータを加工しているか
3. なぜこの設計にしたか

加えて、今後の移行方針として以下を示します。

- Amazon EventBridge
- AWS Lambda
- AWS S3
- Amazon CloudFront
- AWS Athena

## 対象リポジトリと役割

- `ohisama-backend`: データ取得、整形、差分検知、配信用JSON生成
- `ohisama-portal`: フロントエンド本体（JSONを読み込み、検索・フィルタ表示）
- `public-ohisama-portal`: 架空データのみを使った公開用サンプル

## 1. どのようにデータを取得しているか

### TVer

- Playwrightで対象タレントページを巡回し、番組情報を抽出

### YouTube

- YouTube Data APIでチャンネル動画を取得


## 2. どのようにデータを加工しているか

### TVerの加工

- 前回と比較し、新規追加分、配信終了予定日時を基にタグ名を追加する

### YouTubeの加工

- 新規取得データを既存ターゲットデータへマージ
- TypeScript管理データをフロント利用用JSONへ変換

### フロントエンドでの利用

- バックエンドで生成したJSONを読み込み、一覧化、検索、種別・メンバー絞り込みを実行

## 3. なぜこの設計にしたか

結論として、**JSONファイルをテーブル代わりに使う設計**にしています。

背景:

- メンテナンスコストを最小化したかった
- 個人開発として、まず継続運用しやすい形を優先したかった

この設計の実務上の利点:

- DBサーバーの運用が不要（初期構築・保守の負担を抑制）
- Git差分でデータ更新履歴を追いやすい
- スキーマがシンプルで、フロントへの受け渡しが明確
- バッチ処理を小さな責務で分割しやすい（取得、比較、付与、同期）

## 現在の課題

現在はGitHub Actionsで更新していますが、次の課題があります。

- データ更新ログとファイル更新ログのコミットが混在し、管理しづらい

## 今後のアーキテクチャ移行方針（AWS）

今後は、GitHub Actions中心の同期から以下へ移行します。

1. EventBridgeで定期実行をスケジュール
2. Lambdaで取得・整形バッチを実行
3. 生成したJSONをS3へ保存
4. CloudFront経由でフロントからS3のJSONを配信

期待する効果:

- データ更新処理とアプリのコード履歴を分離できる
- リポジトリのコミット履歴が読みやすくなる
- データ配信責務をAWS側に集約できる

## 次の分析ステップ（AWS Athena）

S3に蓄積したJSONを基に、Athenaで分析を実施する計画です。

想定する分析例:

- 更新件数の推移（日次・週次）
- ソース別件数（TVer / YouTube）
- タグ別件数（new / almost-over）
- メンバー別の露出傾向

これにより、表示改善やバッチ最適化をデータに基づいて判断できる状態を目指します。

## 技術スタック

- Next.js 15
- React 19
- TypeScript

- GitHub Actions（現行の自動実行基盤）

## 主要ページ

- `/almost-over`: 期限が近いコンテンツ一覧
- `/all`: 全コンテンツ一覧
- `/:type`: 種別ごとの一覧
- `/:type/member/:member`: メンバー別一覧
- `/about`: サイト情報とサンプル利用上の注意

## セットアップ

1. 依存関係をインストール: `npm ci`
2. 開発サーバーを起動: `npm run dev`

## SEO関連の環境変数

- `NEXT_PUBLIC_SITE_URL`: canonical / Open Graph / sitemap / 構造化データのURL生成に使用

## データ方針（公開用）

- `data/` 配下はすべてサンプルデータ
- メンバー名、番組名、説明文、更新日時は架空
- 外部リンクは `example.com` を利用

## テスト

- Smoke tests: `npm run test:e2e:smoke`
- Full tests: `npm run test:e2e`
- Report: `npm run test:e2e:report`
